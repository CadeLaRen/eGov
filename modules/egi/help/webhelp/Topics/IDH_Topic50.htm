Creating a Module

 

To create a new module, perform the following steps:

1.   Log on to the Administration System.

2.   Click Module from the left navigation bar.

3.   Click Create to display the Module Set up page.

4.   Specify the module details as described in the following table:

Table 1: Module Set Up

Field

Description

Module Name

Enter the name of the module.

Module Name Local

Enter the name of the module in local language.

Module Description

Enter the module name that you want to appear in the user interface.

Parent Module

Do the following steps:

1.   Click the Browse button . You get a Search Screen Module page.

2.   Click the module that you want to add in your ERP Suite.

 

If you want to add a submodule, expand the parent module and select a submodule. For example, if the parent module is Reports and the module that you are creating is Salary Reports, then the module Salary Reports appears under the module Reports.

NOTE: Click the Plus button  to select the submodules.

Is Enabled

Select the check box to activate the module.

Base URL

Enter the URL of server and the port number where the ERP Suite is running. Eg.,192.168.1.44:9980/egi

Order Number

Enter a number to indicate the order in which the module should appear on the UI.

6.   Click Create to save the information.

 

You get a message confirming that the module has been created successfully. The modules that you have created will be listed in the ERP Suite when a user whose role has been mapped to access this module logs in to the ERP Suite.

 

The created modules will not have any features (action) in it. For example, assume that you have a created a submodule called "Create Property" in PTIS. Then, there will not be any links (features) to create a property in the submodule.

 

To include features/actions in a module, you need to set up an action for it.
