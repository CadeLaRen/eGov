Introduction 

 

For effective administration, a city is divided into multiple administrative units.  For example, Chennai is divided into zone, ward, area, locality, and street, while Bangalore is divided into zones, wards, and streets.

 

In other words, in the City of Bangalore, for the "administration" hierarchy type, you have Zones, Wards, and Blocks as boundary types.

 

See the example below:

·         City: Bangalore

·         Hierarchy Type - Administration

·         Boundary Type - ZoneWardBlockStreet

·         Boundary Values- Ward 1, Ward 2, ..Ward n ->Block 1, Block 2, ..Block n->Street 1, Street 2, ..Street n

 

In addition to the administration, there can be a hierarchy type based on each application.

 

See the example below:

·         Hierarchy Type - Birth and Death

·         Boundary Type - CityRegistration Unit

·         Boundary Values- Bangalore->R Unit 1, R Unit 2, ..R Unit n.

 

 

 

Why Do You Set This Up?

 

You need to set up the hierarchy types, boundary types, and boundary values in the system to be able to use them across all modules.

 

·         Assume that you want to create a property in the system using eGov PTIS Chennai Implementation. Since Chennai is divided into zone, ward, area, locality, and street, you must specify the zone in which the property lies, ward, and so on. Since you have already set up these values, you can directly select them from the corresponding drop-down lists.

 

·         When you assign a boundary to a user, his/her jurisdictional power is limited to the specific boundary. From the PTIS perspective, assume that you have mapped "Assessor" to Ward 1. Now, the Assessor will be able to access the properties pertaining to only Ward 1.
