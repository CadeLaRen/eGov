Introduction

 

Chart of accounts are  list of accounts used by a ULB. From a ULB perspective, the chart of account is created for the following accounts:

·      Income: It represents the ULB's gross earnings. Eg: Income from Property Tax.

·      Expenditure: It represents the ULB's expenditures to enable itself to operate. Eg: Expenditure being incurred for electricity and water usage.

·      Liability: It represents the different types of economic obligations by a ULB, such as accounts payable, bank loan.

·      Asset: It represents the different types of economic resources owned by a ULB. Eg: Cash in bank, building, accounts receivable.

 

The chart of accounts are usually set up in a tree structure, and is done as follows:

·      Type:– Income/Expenditure/Liabilities/Assets. For example: Income (code 1), Expenditure (code 2), Liabilities (code 3), and Assets (code 4).

·      Major Head:– Sub-classification within a type. For example: Tax Income (code 110).

·      Minor head:– Sub-classification within a major head. For example: Property Tax (code 11001).

·      Sub-minor head:– Sub-classification within a minor head. For example: Property Tax collected for state government properties (code 1100103)

·      Detailed Code – Sub-classification within a sub-minor head. For example: Property Tax collected for state government properties from a specific zone (code 110010301)

 

All accounting entries must be identified by standard chart of accounts consisting of:

·      Fund Code

·      Function Code – for all transactions involving budgetary codes

·      Source of Financing Code

·      Account Code

 

Transaction posting (posting of vouchers) will be done at the lowest level in the hierarchy (at the detailed codes).

 

NOTE: It is not mandatory to set up chart of accounts with all the levels mentioned above. The number of levels depends on your business requirements.
