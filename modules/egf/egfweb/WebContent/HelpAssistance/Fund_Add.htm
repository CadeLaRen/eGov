Fund add: 

Objective:

The Fund is a defining concept in municipal accounting – where it is required to segregate all accounting transactions into designated funds. Each fund needs to be treated as an independent accounting entity – in other words, all vouchers within a fund must be self-balancing and balance sheets and Income/Expenditure reports must be generated for each fund. The following must be supported:

A hierarchy of funds may be defined – i.e. each fund can have multiple sub-funds and so on.

Each fund must have an identifier – each voucher belonging to a fund must have the identifier embedded in the voucher number for easy identification. Fund is taken at a voucher head level for each voucher transaction. This is a mandatory field in all transactions.

Each fund must have the following two account codes defined:

Receivables code: For inter-fund receivables accounts (to be used when a fund transfers out money to another fund)

Payables code: For inter-fund payables accounts (to be used when a fund receives money from other funds)

Only the ‘leaf-level’ nodes are taken in a transaction. These are funds which do NOT have any sub-funds.

Field Description Table:

Field Name Field Type Field Description
Fund Code Numeric Mandatory 
Fund Name Alphanumeric Mandatory 
Parent Fund Name Alphanumeric Optional. Selection of a parent type will create the fund as a sub-fund (child) of a fund already created (parent fund).  
Active Check Box By clicking on, the Fund becomes active, and accounting of transactions under the fund is enabled. By clicking off, the Fund becomes inactive, and no transactions can be accounted under the Fund.
