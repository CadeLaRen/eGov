Objective: Once the reconciliation is done, the user uses the summary to view the cheques which are all in the bank book but not in bank statement.

Field Description Table:

Field Name Field Type Field Description
Bank And Branch Drop down list List of active bank.
Account Number Drop down list List of account number based on the bank.
Bank Statement Balance Numeric The user can enter the amount from the bank statement.
Bank Statement Date Date The user can enter the date from the bank statement.

Expected Results and/or Workflow:
It will show the cheque details which are all in the bank book but not in bank statement and not yet reconciled.

Action Expected Result
View Summary Onclick of View Summary,Returns all the cheque details in a tabular form which are all in the bank book but not in bank statement and which are all not yet reconciled with the search criteria.
Cancel Cancels the transaction.After that the screen will be cleared.
Close Close the screen.
Refresh Once again it will return the search results with that corresponding search criteria.
Print It will Print the Reconciliation Summary.

Troubleshooting:
Question 1:
If you enter the Bank Statement Balance in non numeric type.
Answer 1:
The system will give alert message like: Enter Numeric value for Bank Statement Balance.
Question 2:
If you enter the wrong date format.
Answer 2:
The system will give alert message like: Invalid date format: Enter Date as dd/mm/yyyy.
