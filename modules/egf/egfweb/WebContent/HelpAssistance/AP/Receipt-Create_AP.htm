Objective: This screen is used for creating Miscellaneous Receipt.

Field Description Table:

Field Name Field Type Field Description
Voucher No. Text Voucher number for the Receipt.If it is auto voucher generation no need to give voucher number.
Counter Drop down list Counters are defined on the Collection Agent Screen.
Voucher Date Date Voucher Date. This date needs to be in an Open Period.
Collection Mode Option button Cash/Cheque/Bank
Bank Option button List of Banks where the ULB has an account under the Fund.
Fund Drop down list List of active funds.
Scheme Drop down list List of Scheme based on the Fund.
Sub Scheme Drop down list List of Sub Scheme based on the Scheme.
Financing Source Drop down list List of Financing Sources.
Field Drop down list List of Fields.
Amount Numeric Amount Received.
Narration Alphanumeric Description.
Cheque No Numeric Cheque number if Mode: Cheque.
Cheque Date Date Cheque Date if Mode: Cheque.
Cheque Amount Numeric Cheque Amount if Mode: Cheque.
Boundary Level Drop-down: List of Administrative boundaries at which accounting is managed Defaulted to the ULB since all cheque-in-hand/cash accounts are defined at the ULB level
Cheque-in-hand/Cash-in-hand Account Defaulted from setup Account defined from the Setup Cheque-in-hand/Cash-in-hand Screen.
Function Text The function is set to required for the Account head.
Account Code Text The Account code is set to required for the Account head.
Account Head Display From the Chart of accounts.
Dr Amount Numeric Debit Amount.
Cr Amount Numeric Credit Amount.
Click Additional details Only if additional details are required for any control account code.

Expected Results and/or Workflow:
Create the receipt for the particular income code.

Action Expected Result
Save& New Submits the transaction. On successful submit, the system generates the Receipt Voucher Number. The same screen is made available for entering a new record.
Save&Close Submits the transaction. On successful submit, the system generates the Receipt Voucher Number. The screen is closed.
Save&View Submits the transaction. On successful submit, the system generates the Receipt Voucher Number and it will open new screen in view mode with transaction details.Here Exit and Print option is available.
Cancel Cancels the transaction.
Show GL Entry Shows the entry that will passed to the GL on submitting the transaction.

Troubleshooting:
Question 1:
If you enter the Amount less/greater than the sum of total credit amount.
Answer 1:
Difference between Total Cr Amount and Total Dr Amount should be equal to the Amount,this alert message will come.
