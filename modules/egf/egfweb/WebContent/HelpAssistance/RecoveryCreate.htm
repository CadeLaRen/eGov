Recovery Create: 

Objective:

The Recovery types that are created will be mapped to the above Work/Contract types and Party types. Hence any Invoice that is made for that work type WO and Party type will list only those Recovery types that are mapped to the work type and Party type. Every deduction/Recovery in the invoice should be made using the “Recovery type” that is defined in the master. Any deduction that is entered without the deduction type in the Invoice cannot be remitted using remittance module.

Field Description Table:

Header Section:

Field Name Field Type Field Description
Recovery Code User Entry The user defined  code for recovery Type.
Recovery Name Text The name for the recovery code created.
Account Code Select List The Ledger account code that will be credited for the deduction amount in the invoice. When the Recovery/deduction type is selected in the invoice, the account code will be automatically retrieved.
Applied To Select List  The type of party to which the deduction/recovery is linked. This means, the defining recovery code can be selected only in those invoice that is done for the party who is mapped to the type selected here.
Remitted To Text  Enter the name of the party/Gov body/institution to which the deduction made will be remitted later using remittance module
BSR Code Text  Enter BSR code as applicable if it is remitted into the particular deduction’s bank account.

Detail Section:

Field Name Field Type Field Description
Party Type Select List Specific Sub type (if it is applicable only to the sub type).
Doc Type Select List The type of work to which the deduction/Recovery is linked. This means, the defining recovery code can be selected only in those invoice that is made for the WO, which is of work/contract type selected here.
Doc Sub type Select List Specific Sub type (if recovery is applicable only to the sub type).
Date (From/To) Date Date range between which the recovery code is valid. The recovery code will not be shown in the deductions list of invoices before or after these dates.
Payment Limit (Low/High) Number The invoice amount-range for which the Recovery code are applicable.
Percentage (IT /Sur Charge/Edu Cess/Total) Number Percentage of various components that should applied in the invoice when this recovery code is selected. Total will show the total of all the components entered. The total percentage is considered at the invoice to calculate the deduction amount.
Flat Amount Number If the recovery is for a definite amount, then the flat amount can be entered instead of the percentages.
Add Row Button Click button to add a row in which a new date range, payment limit and percentage can be entered for the same recovery code. If a deduction’s percentage is varying from one quarter to another, it can be configured in the new row or even if the percentage is differing based on the invoice amount, it can be configured as a new row.
Delete Row Button Click this button to delete a row in the grid.

Expected Results :

Action Expected Result
Save & Close Submits the transaction. On successful submit, the system creates the recovery code and the screen is closed.
Save & New Submits the transaction. On successful submit, the system creates the type and the same screen is made available for entering a new record
Cancel Cancels the transaction and loads the page afresh.
Close Closes the screen.
