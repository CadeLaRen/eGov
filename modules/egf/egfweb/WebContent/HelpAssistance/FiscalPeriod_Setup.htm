Closing Balance: 

Objective: Period closing is an activity that must be performed to ‘close’ a period or a financial year.

The following operations must be allowed:

Close a Financial Year and Transfer Closing Balance for a specific Financial Year: This does not allow any postings for that Year (update the Active for Posting Flag to 0) and computes the closing balances as follows:

 Income/Expenditure Accounts: For each income account (Type=1), the net balance is computed as follows: Closing balance = Opening credit balance + Sum of credit transactions – (Sum of debit transactions + Opening debit balance)

For each expenditure account (Type=2), the net balance is computed as follows: Closing balance = Opening debit balance + Sum of debit transactions – (sum of credit transactions + Opening credit balance)

The net balance (excess of income over expenditure) is computed once the above is done:

If the balance is greater than zero, the amount is added to the CREDIT BALANCE of the liability account that is assigned a purpose code of 7 (‘ExcessIE’) If the balance is less than zero, the amount is added to the DEBIT BALANCE of the liability account that is assigned a purpose code of 7 (‘ExcessIE’). Only one liability detailed account code MUST be designated with purpose code of 7. The net balance is updated as the opening balance of the NEXT FINANCIAL YEAR for that liability account code.

Asset/Liability accounts: For each liability account (Type=3), the net balance is computed as follows: Closing balance = Opening credit balance + Sum of credit transactions – (Sum of debit transactions + Opening debit balance)

For each asset account (Type=4), the net balance is computed as follows: Closing balance = Opening debit balance + Sum of debit transactions – (sum of credit transactions + Opening credit balance) For each asset/liability account, the net balance is updated as the opening balance of the NEXT FINANCIAL YEAR for that account code.

Transfer Closing Balance:Performs the transfer operation (as detailed above)

Close Date Range: Closes the date range and does NOT allow any transactions to be posted to that date range (ie voucher date CANNOT lie within that range). If the HARD CLOSE flag is selected, the date range cannot be opened again.

Re-open date range:Re-opens a date range that has been closed earlier so that vouchers can be posted within that date range. However, the Financial Year status will override this condition. In other words, if the Financial Year has a status of Active For Posting =0, then re-opening of a date range will NOT allow any vouchers to be posted for that date range.
