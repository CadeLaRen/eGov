Opening Balance: 

Objective: Opening balances need to be setup during the first year of operation (when historical data will be migrated into the system). On an ongoing basis, opening balances for any year will be automatically calculated by the system – however, in case any new opening balances need to be set (e.g. when an old asset that was earlier not in the books is valuated by the ULB), then this screen may be used.

Field Description Table:

Opening Balance Header:

Field Name Field Type Field Description
Fund Drop-down From the fund master
Financial Year* Drop-down From the Financial Year master
Type Drop-down Asset / Liability  
Major Head Drop-down List of major codes 
Minor Head Drop-down  List of minor codes 
Financing Source Drop-down  List of financing source master 

* All financial years defined in the master which are Active can be chosen.

* All major codes (classification=1) under the Type will be shown

* All minor codes (classification=2) under the Major Code will be shown if major code is selected. If no major code is selected, ALL minor codes under the Type will be shown

Opening Balance Details:

Field Name Field Type Field Description
Account Code Drop-down From the Chart of Accounts – list is restricted based on the type, major and minor heads selected above
Account Head Display  Account description
Accounting Entity Drop-down List of sub-ledger codes assigned to the Account Code 
Debit Amount* Numeric  
Credit Amount* Numeric   

* For an account code, only a credit OR a debit opening balance may be setup.The list (i.e. sub-ledger codes) is from the values assigned under the accounting entity mapped to the account code.

Expected Results :

Action Expected Result
Submit Opening Balance Set up  will be created.
Cancel Refreshes the screen .
Close Closes the screen.
