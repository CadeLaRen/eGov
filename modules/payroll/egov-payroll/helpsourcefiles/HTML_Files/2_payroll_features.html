General Software Features

The key aspects of the eGov application are presented in this chapter.

Platform

The eGov application is part of the Integrated Municipal e-governance Application Suite developed by eGovernments Foundation. Full integration of all the modules is depicted in the figure below . All the modules are developed as web-based applications and can be accessed over the Internet using a web-browser. This has the following advantages:

  • Ease of Installation: Since the client machine needs to have only the ubiquitous web browser, no software installation is required on the client machines.
  • Multi-user, Integrated Database System: The architecture and application design is built ground up for usage by multiple users who might be located geographically in different offices. All the users in the system have the same view of the system in real time.
  • Integration Platform: The eGov application is part of an Integrated Architecture that brings together all the departments within a ULB into a common information platform.
  • Proactive Information Disclosure: Availability of the data on an Internet platform allows for easy integration into a proactive information disclosure platform (for example, city websites), providing for an easy implementation of the Right To Information (RTI) Act.

Working on Multiple Windows

The application works in a web-browser window (Internet Explorer) and allows the user to open multiple windows at a time. From the main menu screen, the user can open a new window using the Ctrl+N key (alternatively: File -> New -> Window). This helps the user to access information from other screens or modules while working on a particular screen or module.

Selection from Drop-down Lists

The application provides drop-down lists wherever possible, allowing the user to select from the lists. For example, in the View Advance screen, when you type the employee code a list of employee codes is displayed. When a given employee code is selected, the Employee Name field displays the name of the employee.

Drop-down lists are of three types:

Click on the arrow mark to display the options available. This is used when the list of drop-down options is not very long.

Click to search for relevant values. This is used when the list of drop-down options is long. Data can be filtered based on the value entered in the text box.

Type the value in the field. When you start to enter the number, it displays the values that begin with the number you typed. Click to select the record from the list.

Screen Help

All transaction screens are provided with an icon with a question mark on the top-right corner (see below) that takes the user to the screen level help.

When the help button is clicked, the help window pops up. This provides you with an overview of the screen as well as help with all the fields on the screen.

Voucher Numbering

Voucher numbers need to be manually entered for each voucher. Manual voucher numbers are entered based on the following rules:

  • The first letter identifies the Fund to which the Voucher belongs. This identifier is assigned to the Fund as part of Setup.
  • The second letter identifies the type of voucher. The options are: Receipts, Payments, Contras, and Journal Vouchers.
  • This is followed by a number (up to 10-digits).

The eGov application also generates a Computer Generated (CG) number internally for each voucher. The CG number can be used by users to retrieve specific vouchers at any point in time.

NOTE: The voucher numbering remains unique since it is based on the financial year and the voucher type. The CG number is re-set to 00001 at the beginning of each financial year.

On successful completion of a transaction, the following message is displayed: Transaction Successfully executed. It also displays the following information:

  • Voucher number entered by the user
  • CG number generated by the system

NOTE: In case of a system error, it is strongly recommended to record the error message and report it to the support desk. The support desk will help resolve the problem.

Built-in Calendar

eGov applications use a built-in calendar to validate the voucher date for each transaction. For example, the following conditions are checked in the Create Employee screen for eGov EIS Payroll:

1. The Date of first Appointment of an employee should be before the From Date in the Assignment Details section.

2. In the Assignment Details section the To Date of one assignment should be prior to the From Date of the next assignment.

3. The employee's age is validated so that the minimum age of the employee is 15 years.
