System Overview

This chapter describes how to access eGov applications and view the different menu options. The menu options that are displayed depend on the role that has been assigned to the user.

User Roles

A user role is assigned to each employee of a ULB, when an employee record is created. What is important to note is that a user role for the purpose of working with eGov applications is different from the employee designation within the ULB. To illustrate, an employee with a designation of Senior Accountant could have a user role Super User. With this user role, the employee would have wide access to all the features of the application.

Logging on to the Application

eGov applications are web-based and can be accessed from a Web browser. The system can be accessed either by typing the URL of application in the Address bar or by accessing the site from the city Web site. When you type the URL in the Address bar, the login page is displayed.

Login Screen

Each user is provided a unique UserName and Password.

1. In the UserName field, type your UserName.

2. In the Password field, type your password.

3. Click Login.

The system then validates these details. When a valid UserName and Password combination is entered, the application home page is displayed.

The home page is divided into the following parts:

  • The Header Section which shows the following:
    • Name of the ULB
    • eGovernments logo. If a user clicks on this when in any other screen, they return to the home page.
    • Logout link. Click this when you have finished your work and wish to exit from the system.
  • Menu Tree Section: This lists the menu in a tree form. The actual contents of the tree are populated based on the role assigned to a particular user. eGov applications are fully menu-driven with a multi-level menu structure which allows a user to select an option in the menu and go to the next lower level with each selection.
