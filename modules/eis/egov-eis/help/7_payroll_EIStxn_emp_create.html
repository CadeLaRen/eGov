Create Employee

Employee details are entered in the Create Employee screen to create an employee record.

1. Enter the personal details of the employee in the Employee Personal Details section. The table contains field descriptions.

NOTE: All field names that are followed by an asterisk are mandatory, values must be provided in them.

Field Description
Employee Code Automatically generated for a new employee.
Date Of Birth* Type the employee's date of birth in the (dd/mm/yyyy) format. For example, if the date of birth is 1st February 1980 type 01/02/1980.
First Name* Type the first name of the employee.
Middle Name Type the middle name of the employee.
Last Name Type the last name of the employee.
Father/Husband's First Name Type the first name of the employee's father or husband.
Middle Name Type the middle name of the employee's father or husband.
Last Name Type the last name of the employee's father or husband.
Gender Click the button beside the gender of the employee.
Blood Group To select the blood group of the employee, click the relevant option in the drop-down list.
Mother Tongue Type the mother tongue of the employee.
Religion To select the religion of the employee, click the relevant option in the drop-down list.
Community To select the community of the employee, click the relevant option in the drop-down list.
Is Physically Handicapped? Click Yes if the employee is physically handicapped.

Click No if the employee is not physically handicapped.
Is Medical Report Available? Click Yes if the medical report of the employee is available.

Click No if the medical report of the employee is not available.
Identification Mark-1 If the employee has an identification mark (such as a birthmark or scar), type a description of this.
Identification Mark-2 If the employee has another identification mark, type a description of this.
PAN Number Type the PAN number of the employee.
Address/Permanent Address:* Address Line-1 and Address Line-2 Type the permanent address of the employee.
Address/Correspondence Address: Address Line-1 and Address Line-2 Type the employee's address for correspondence.
Languages Known To select the language known to the employee, click the relevant option in the drop-down list. To select multiple languages, press the SHIFT key and click on all the languages known to the employee.
Local Language Qualified Click the local language in which the employee's qualification degree was attained.

2. Enter the employment details of the employee in the Employee Employment Details section.

The table contains field descriptions.

Field Description
Status To select the status of the employee, do one of the following:
  • Click Employed when the employee is currently employed at the ULB (in service). A payslip needs to be generated for such employees.
  • Click Deceased when the employee is deceased.
  • Click Retired when the employee has retired from the services of the ULB. Such an employee is eligible to receive a pension.
  • Click Suspended when the employee is suspended. Such employees will not have a payslip generated.
Date of First Appointment* Type the date of appointment of the employee in the (dd/mm/yyyy) format. For example, if the date of appointment is 1st February 2005 type 01/02/2005.
Employee Status To select the status of the employee, click the relevant option from the drop-down list. For example, active, temporary, and so on.
Mode of Recruitment* To select the method of recruitment of the employee, click the relevant option from the drop-down list. For example, direct recruitment, transfer, and so on.
Recruitment Type* To select the recruitment type of the employee, click the relevant option from the drop-down list. For example, staff.
Category To select the category of work for the employee, click the relevant option from the drop-down list. For example ministerial, medical, and so on.
GPF Account No Type the General Provident Fund account number of the employee.
Pay Fixed In To select the pay commission, click the relevant option from the drop-down list. The pay commission is based on the employee payscale.
Grade* To select the grade assigned to the employee, click the relevant option from the drop-down list.
Retirement Age Displays the retirement age based on the grade assigned to the employee.
Retirement Date Displays the proposed date of retirement of the employee in the (dd/mm/yyyy) format. The date can be changed, if required.

3. Enter the payscale details of the employee in the Payscale Details section.

The table contains field descriptions.

Field Description
Payscale Name* To select the payscale applicable to the employee, click the relevant option from the drop-down list.

NOTE: To add another payscale name, click Add Row.
Effective Date* Type the date from when the payscale is effective from, in the (dd/mm/yyyy) format. For example, if you type 01/02/2006, the selected payscale is used during payroll calculation from 1st February 2006.
Annual Increment Date* Type the date from when the annual increment is effective from, in the (dd/mm) format. For example, if you type 26/02, the annual increment is added every year in the month of February.
Basic From Displays the minimum amount that should be paid to the employee. This is based on the selected payscale name.
Basic To Displays the maximum amount that can be paid to the employee. This is based on the payscale name selected.
Basic Pay* Type the employee's basic pay. This value should be within the Basic From and the Basic To range.

4. Enter the account details of the employee in the Account Details section.

The table contains field descriptions.

Field Description
Bank Name* To select the bank where the employee has an account, click the relevant option in the drop-down list.
Branch Name* To select the branch of the bank where the employee has an account, click the relevant option in the drop-down list.
Account Number* Type the account number of the employee.
Salary Account Click Yes if the account number is a salary account. Click No if the account number entered is not a salary account.

5. Enter the user name details in the User Details section.

The table contains field descriptions.

Field Description
Is User Active Click Yes to activate the user name of the employee.
User Name* Enter a user name for the employee. This is one of the requirements for the employee to log in to the eGov application. However, the employee also needs to be given a user role in the Admin Module.

To log in with the above user name, the password needs to be in the format, employee first name underscore employee code. So if the employee's first name is Usha and her employee code is 1190, her password would be: Usha_1190

The password can be changed in the Admin Module.

6. In the Assignment Details section, enter the details related to the designation assigned to the employee.

The table contains field descriptions.

Field Description
If Head of Department Click Yes if the employee is assigned the Head of the Department post.

Click No if the employee is not assigned the Head of the Department post.
From Date* Type the start date from when the employee reports for the assigned post. This date should be after the appointment date.

From this date all the details that are relevant to the new post will be used for payslip calculations.
To Date* Type the last date till when the employee needs to report for the assigned post.
Fund* To select the name of the fund the employee is assigned to, click the relevant option from the drop-down list. This fund is debited during salary payment to the employee.
Function* To select the functional area assigned to the employee, click the relevant option from the drop-down list.
Designation* To select the designation assigned to the employee, click the relevant option from the drop-down list.

NOTE: To add another designation, click Add Row.
Position* To select the position assigned to the employee, click the relevant option from the drop-down list. A designation can have many positions associated with it. For example, for a Senior Engineer designation, positions can be Senior Engineer Area 1, Senior Engineer Area 2, and so on.

The position is used to determine approval hierarchy in workflows. For example, when a work order is submitted by a Junior Engineer, it is sent for approval to the Assistant Engineer, and then to the Senior Engineer. Since the system is based on position as opposed to individuals, work is not affected when an employee retires or is transferred.

NOTE:A single employee can be assigned to a particular position for a specific period. When you try to assign a position that is already assigned to another employee within that time period, an error message is displayed.

A new position for a designation can be added in the Admin Module.
Functionary* To assign the employee to a functionary, click the relevant option from the drop-down list.
Departments* To select the departments the employee works in, click the relevant option from the drop-down list. To select multiple departments, press the SHIFT key and click on the relevant departments.

NOTE: This field is displayed only if the employee is the Head of Department.

HOD* To select the departments for which the employee is the Head of Department, click the relevant option from the drop-down list.To select multiple departments, press the SHIFT key and click on all the departments.

The options in this field are displayed in the HOD drop-down list based on the employee Departments chosen.

NOTE: This field is displayed only if the employee is the Head of Department.

Main Department* Click the name of the main department to which the employee belongs from the drop-down list. If the employee is the Head of Department, click the name of the department the employee reports to, from the drop-down list.

7. After you have filled in the necessary details, click:

  • Save to create a new employee account.

    Or

  • Close to clear all the details and close the screen. A new employee account is not created.
